Curated notices for components that are redistributed inside docs-sign but whose
license texts are NOT carried in the Go module cache or in node_modules. These are
concatenated verbatim into THIRD_PARTY_LICENSES by scripts/gen-third-party-licenses.sh.

The dominant entry here is PDFium: the package github.com/klippa-app/go-pdfium embeds a
precompiled PDFium build (pdfium.wasm, via //go:embed) which is therefore baked into the
docs-sign binary and shipped in the container image. The klippa Go module ships only its
own (MIT) license — PDFium's own license and the licenses of the third-party libraries
PDFium bundles must be reproduced separately, which is what this file does.

Third-party libraries bundled inside the PDFium build
--------------------------------------------------------------------------------

The PDFium WebAssembly build links additional third-party libraries, each under its
own permissive license. The authoritative per-library license texts live in the
PDFium source tree under third_party/<lib>/LICENSE:
https://pdfium.googlesource.com/pdfium/+/refs/heads/main/third_party/

Container base image
================================================================================

The published container image is built FROM ghcr.io/linuxserver/baseimage-alpine
(LinuxServer.io), which itself bundles Alpine Linux packages and s6-overlay. Those
components remain under their own licenses, whose texts are shipped inside the
image (e.g. under /usr/share/licenses and the s6-overlay distribution). s6-overlay
is distributed under the ISC license; Alpine base packages carry a mix of
permissive and copyleft licenses as provided by the Alpine project. See:
https://github.com/linuxserver/docker-baseimage-alpine
https://github.com/just-containers/s6-overlay
